# 生态网络分析流程说明

## 概述

本项目用于基于最小累积阻力模型（MCR）构建生态网络和生态廊道。分析流程包含 4 个步骤，从原始生态源地数据开始，最终生成生态网络（GraphML）与廊道/节点矢量数据（Shapefile）。

本项目提供两套实现：
- 脚本版：按步骤运行 4 个主脚本，输出到 `OutputData`，适合科研复现、批处理与并行计算
- BuildEXE 版：GUI + EXE 打包版（见 `BuildEXE/`），适合交付与非开发用户使用

两套实现的核心算法一致，差异主要在输出文件、依赖与并行化支持（详见文末“脚本版 vs BuildEXE 版”）。

## 输入数据

### 原始输入数据（位于 `InputData` 目录）

1. **EcologicalSource.tif** - 生态源地栅格数据
   - 二值栅格（0/1）
   - 1表示生态源地

2. **Connectivity.tif** - 属性栅格数据（建议使用连通性栅格）
   - 用于廊道提取的权重/成本面

3. **ResistanceSurface.tif** - 阻力面栅格数据
   - 物种穿越不同景观单元的阻力系数

## 工作流程

```
EcologicalSource.tif
        ↓
    [步骤0]
预处理工具
        ↓
    [步骤1]
邻接关系计算
        ↓
    [步骤2]
激活点提取
        ↓
    [步骤3]
生态廊道构建 ← Connectivity.tif (属性栅格，建议采用连通性)
                ResistanceSurface.tif (阻力面)
        ↓
    最终结果
```

## 详细步骤

### 步骤0：预处理工具 (0_PreprocessTool.py)

**功能：**
- 栅格转矢量面
- 过滤小斑块（面积阈值：1 km²）
- 面部件消除（孔洞填充）
- 提取斑块中心点
- 矢量数据栅格化

**输入：**
- `InputData/EcologicalSource.tif`

**输出（位于 `OutputData` 目录）：**
- `mspa_core.shp` - 源地矢量面
- `mspa_core_center.shp` - 中心点矢量
- `mspa_core_filled.tif` - 面部件消除后的源地栅格
- `mspa_core_center.tif` - 斑块中心点栅格

**运行命令：**
```bash
conda activate geobase
python 0_PreprocessTool.py
```

---

### 步骤1：邻接关系计算 (1_AdjacentCore.py)

**功能：**
- 计算源地斑块之间的邻接关系
- 欧氏分配（Euclidean allocation）：将背景像元分配给最近斑块
- 邻接提取：通过栅格位移（8 邻域）向量化扫描提取斑块邻接对（避免逐斑块两两判断，速度更快）

**输入：**
- `OutputData/mspa_core_filled.tif`

**输出（位于 `OutputData` 目录）：**
- `mspa_core_filled_eucall.tif` - 欧几里得分配栅格
- `adjacency_parallel.csv` - 邻接关系表

**运行命令：**
```bash
conda activate geobase
python 1_AdjacentCore.py
```

---

### 步骤2：激活点提取 (2_GetActivateP.py)

**功能：**
- 获取斑块边界
- 计算相邻斑块之间的最近点对
- 提取廊道激活点位置

**输入：**
- `OutputData/mspa_core_filled.tif`
- `OutputData/adjacency_parallel.csv`

**输出（位于 `OutputData` 目录）：**
- `adjacency_parallel.csv` - 更新为包含最近点对信息（追加列）：
  - `NearestP_Block1_X / NearestP_Block1_Y`
  - `NearestP_Block2_X / NearestP_Block2_Y`
  - `Distance`

说明：脚本版当前不再单独输出激活点栅格文件（减少大文件 I/O）；激活点信息以表格形式保存在邻接表中，供步骤 3 使用。

**运行命令：**
```bash
conda activate geobase
python 2_GetActivateP.py
```

---

### 步骤3：生态廊道构建 (3_GetCorridorAN.py)

**功能：**
- 基于最小累积阻力模型(MCR)提取廊道
- 使用Dijkstra算法计算成本距离
- 构建生态网络图
- 生成廊道和节点矢量数据

**输入：**
- `OutputData/mspa_core_center.tif` - 源地中心点
- `OutputData/mspa_core_filled.tif` - 源地栅格
- `InputData/Connectivity.tif` - 连通性栅格
- `InputData/ResistanceSurface.tif` - 阻力面
- `OutputData/adjacency_parallel.csv` - 邻接关系

**输出（位于 `OutputData` 目录）：**
- `ecological_network.graphml` - 源点网络图
- `ecological_network_act.graphml` - 激活点网络图
- `ecological_network.shp` - 廊道矢量数据
- `ecological_network_point.shp` - 节点矢量数据

**并行化支持（脚本版）：**
- 步骤 3 支持多进程并行计算，每对相邻斑块独立跑一次 Dijkstra（MCR 路径）
- 可通过环境变量控制：
  - `ELT_N_WORKERS`：并行进程数（默认：自动取 `CPU-1`，上限 8）
  - `ELT_MAX_PAIRS`：限制最多计算的邻接对数量（用于快速测试/抽样验证）

**运行命令：**
```bash
conda activate geobase
python 3_GetCorridorAN.py
```

```bash

python 3_GetCorridorAN.py
```

## 注意事项

1. **步骤顺序**：必须按0→1→2→3的顺序执行，后续步骤依赖前置步骤的输出
2. **路径设置**：所有代码已配置为使用绝对路径，指向 `D:\2_HaoweiPapers\1_SOCIAndEco\Ecological-Linkage-Tool-main`
3. **输出目录**：所有输出文件自动保存到 `OutputData` 目录
4. **处理时间**：步骤3最耗时；脚本版支持多进程并行（`ELT_N_WORKERS`），可显著缩短总耗时

## 脚本版 vs BuildEXE 版（差异与并行化）

### 入口与运行方式

- 脚本版：直接运行 4 个主脚本（适合科研复现/批处理）
  - [0_PreprocessTool.py](file:///d:/2_HaoweiPapers/1_SOCIAndEco/Ecological-Linkage-Tool-main/0_PreprocessTool.py)
  - [1_AdjacentCore.py](file:///d:/2_HaoweiPapers/1_SOCIAndEco/Ecological-Linkage-Tool-main/1_AdjacentCore.py)
  - [2_GetActivateP.py](file:///d:/2_HaoweiPapers/1_SOCIAndEco/Ecological-Linkage-Tool-main/2_GetActivateP.py)
  - [3_GetCorridorAN.py](file:///d:/2_HaoweiPapers/1_SOCIAndEco/Ecological-Linkage-Tool-main/3_GetCorridorAN.py)
- BuildEXE 版：GUI 程序入口为 [main.py](file:///d:/2_HaoweiPapers/1_SOCIAndEco/Ecological-Linkage-Tool-main/BuildEXE/main.py)，内部调用 `BuildEXE/step0.py ~ step3.py`

### 输出差异

- 脚本版（`OutputData/`）
  - 仍输出 `mspa_core_center.tif` / `mspa_core_center.*`，并以其作为步骤 3 的节点输入
  - 步骤 2 不再输出激活点栅格，最近点对写入 `adjacency_parallel.csv`
- BuildEXE 版（用户选择输出目录）
  - 不输出 `mspa_core_center.tif`、`mspa_core_activate.tif`、`mspa_core_center.*`
  - 会自动生成 `数据说明.md`

### 并行化支持差异（重点）

- 脚本版
  - 步骤 2：最近点对计算支持多线程（加速批量 `cdist`）
  - 步骤 3：廊道构建支持多进程并行（对 2000+ 邻接对提升明显）
- BuildEXE 版
  - 当前以稳定与可打包为优先，步骤 3 仍为单进程/单线程（但已加入日志进度条）
  - 若要在 EXE 版启用并行，需要同时处理 PyInstaller 打包与 Windows 子进程启动方式，因此暂未启用

## 输出结果说明

### 廊道类型

- **T型廊道**：两个源地之间的直接廊道
- **S型廊道**：源地到激活点的廊道

### 节点类型

- **S（Source）**：生态源地中心点
- **A（Activate）**：廊道激活点（踏脚石）

### 属性字段

**廊道属性：**
- `fromnode`：起始节点ID
- `tonode`：终止节点ID
- `weight`：路径上的平均连通性/成本值（来自 `Connectivity.tif`）
- `sum_weight`：路径上的连通性/成本累积和（来自 `Connectivity.tif`）
- `len`：廊道长度（像素数）
- `type`：廊道类型（T或S）

**节点属性：**
- `nodeid`：节点ID
- `sourceid`：所属源地ID
- `type`：节点类型（S或A）
